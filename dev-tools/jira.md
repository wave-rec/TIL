# Jira란 무엇인가?

- Jira는 Atlassian에서 개발한 이슈 및 프로젝트 관리 도구로 주로 소프트웨어 개발 프로젝트에서 많이 사용되며, 애자일(Agile) 방법론을 지원하는 기능을 포함하고 있음

## Jira의 기능들

**이슈 추적**
- 버그, 작업, 개선사항 등의 이슈를 관리할 수 있는 기능 
- 이를 통해 각 이슈의 상태, 우선순위, 책임자를 명확하게 추적할 수 있음

<br/>

**프로젝트 관리**
- 프로젝트의 목표, 마일스톤, 작업 할당 등을 관리할 수 있음
- 프로젝트의 진척 상황을 시각화하고, 필요한 조치를 신속하게 취할 수 있음
    - 애자일 보드: 칸반(Kanban) 보드와 스크럼(Scrum) 보드를 제공하여 스프린트를 계획하고 진행 상태를 추적할 수 있도록 지원함. 작업의 흐름을 효율적으로 관리할 수 있다는 이점.
    - 보고서 및 대시보드: 보고서와 대시보드를 제공하여 프로젝트의 진행 상황, 팀의 성과 등을 시각적으로 분석할 수 있도록 도와줌. 관리자가 프로젝트의 상태를 한눈에 파악하고, 필요한 결정을 내릴 수 있도록 도움.

<br/>

**통합 및 확장성**
- Confluence, Bitbucket, GitHub 등 다양한 도구와 통합이 가능. 플러그인과 애드온을 통해 기능을 확장할 수 있음.

## 백로그
- 아직 스프린트에 넣지 않은 Task들이 쌓이는 곳

## 스프린트
- 이번 주에 해야하는 할 일들이 묶여있느 곳

## 보통의 워크플로우
### Step 1. Task 만들기
1. 백로그 화면에서 만들기 버튼 클릭
2. 이슈 타입을 Task로 선택
3. Summary에 어떤 일을 할지 입력
4. 미리 Epic을 만들어두고 해당 Epic을 연결
5. 해당하는 컴포넌트를 선택
6. Assignee로 담당자 선택
 
- description에 체크리스트 만드는 법
- [ ] 로그인 폼 디자인 확인
- [ ] Input 컴포넌트 구현

<br/>

### Step 2. 스프린트 만들고 Task 넣기
1. 백로그 화면 위쪽에 Create Sprint 버튼 클릭
2. 스프린트 이름 정하기 (ex 1월 3주차)
3. 백로그 영역에 있는 Task 드래그하여 스프린트 박스 안으로 끌어놓기
4. Task가 충분히 들어가면 Start Sprint 버튼 클릭
5. 기간 설정 후 Start 클릭

### Step 3. 스프린트 진행하기
1. 이제 Board 화면으로 이동
2. 작업 시작할 Task를 To do에서 In progress로 드래그
3. 완료되면 Done으로 드래그

### Step 4. 브랜치 연결하기
- Task 상세화면에서
    - Development 섹션의 Create Branch 클릭
    - 혹은 Git에서 브랜치명에 Jira 이슈 번호를 포함하여 브랜치를 생성하면 연결됨
