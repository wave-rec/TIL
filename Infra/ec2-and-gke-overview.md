# Amazon EC2 인스턴스 & Google Kubernetes Engine (GKE) 개요

클라우드에서 컴퓨팅 리소스를 쓰는 대표적인 방식 두 가지를 정리함.  
EC2는 가상 서버(인스턴스) 단위, GKE는 Kubernetes 기반 컨테이너 오케스트레이션 플랫폼임.

---

## 1. Amazon EC2 인스턴스

- **EC2 인스턴스**는 AWS 클라우드 상의 가상 서버임
- 인스턴스 **시작**부터 **종료**까지 전 과정을 사용자가 제어할 수 있음
- OS 선택, OS/앱 설정, 업데이트 설치 등 사용자 지정이 가능함

### 인스턴스 유형과 비용

- 애플리케이션에 맞는 **컴퓨팅, 메모리, 스토리지, 네트워크** 성능을 제공하는 다양한 인스턴스 유형이 있음
- **사용한 만큼만 과금**됨
  - 인스턴스가 **실행(running)** 상태일 때만 요금 발생
  - **중지**하면 요금 중단, **다시 시작**하면 요금 재개
  - **종료**하면 요금 청구 종료

### 활용 기능

- **EC2 Fleet / Auto Scaling**: 부하에 따라 인스턴스 수를 늘리거나 줄여 성능·비용 최적화
- **스팟 인스턴스, 절감형 플랜**: 비용 절감용 옵션
- **관리형 인스턴스**: EKS Auto Mode 등 서비스가 관리하는 인스턴스는 **관리형** 필드가 `true`로 표시되며, 사용자가 직접 설정을 바꿀 수 없음

---

## 2. Google Kubernetes Engine (GKE) 개요

- **GKE**는 Kubernetes 오픈소스 컨테이너 오케스트레이션 플랫폼의 **관리형 구현**임
- Google Cloud 인프라 위에서 컨테이너화된 애플리케이션을 대규모로 배포·운영할 수 있음

### GKE 구조

- **노드**: Compute Engine VM으로, 여러 노드가 모여 **클러스터**를 이룸
- **워크로드**: 앱을 컨테이너로 패키지하고, 이를 **포드** 단위로 노드에 배포함
- **컨트롤 플레인**: Kubernetes API 서버 등 시스템 구성요소를 실행하는 관리 노드 집합이며, GKE에서 Google이 관리함
- Kubernetes API로 워크로드의 관리·확장·모니터링을 수행함

### 작업 모드

| 모드 | 설명 |
|------|------|
| **Autopilot** | 워커 노드까지 Google이 관리. 포드 기준 비용, 프로덕션 준비 환경 제공. |
| **Standard** | 노드 풀·클러스터를 사용자가 직접 구성·관리. 세밀한 제어가 필요할 때 사용. |

### GKE의 이점 (요약)

- **플랫폼 관리**: Autopilot의 완전 관리형 노드, 출시 채널 기반 자동 업그레이드, 노드 자동 복구, Fleet·Connect 게이트웨이, 로깅·모니터링, Cloud Build·Cloud Deploy 등 CI/CD 연동
- **보안**: 강화된 노드 OS, 정책 컨트롤러, 보안 상태 대시보드, Observability 연동
- **비용**: Autopilot은 실행 중인 포드가 요청한 컴퓨팅만 과금; Spot 포드로 내결함성 워크로드 비용 절감 가능
- **가용성**: 월 99% 이상 업타임 SLO, 고가용성 컨트롤 플레인·리전 클러스터 지원

### 사용 사례

- AI/ML 워크로드, 대규모 데이터 처리, 확장 가능한 웹/게임 서비스, 트래픽 변동에 대응하는 안정적인 애플리케이션 등

---

## 3. EC2 vs GKE (간단 비교)

| 구분 | EC2 | GKE |
|------|-----|-----|
| 단위 | 가상 서버(인스턴스) | 컨테이너(포드) / 클러스터 |
| 관리 수준 | 인스턴스·OS·앱 직접 관리 | Kubernetes 기반, 컨트롤 플레인·(Autopilot 시) 노드 관리형 |
| 적합한 경우 | 단일 VM, 레거시, 세밀한 OS 제어 | 컨테이너 기반 앱, 오토스케일, 멀티 서비스 배포 |
